# Этап 03. Параметры, формы, query и POST

## Учебная цель

Показать работу с входными данными запроса:

- HTTP method;
- путь;
- строка запроса;
- данные формы из `POST`.

Поток данных:

`браузер -> HTTP-запрос -> handler -> разбор query/формы -> ответ`

## Как запустить

Из каталога этапа:

```bash
go run .
```

Из корня репозитория:

```bash
make run-03
```

## Какие URL открыть

- `http://localhost:18083/`
- `http://localhost:18083/?name=Алиса`

## Что демонстрирует приложение

- `GET /` показывает страницу с формой;
- query-параметр `name` читается из URL;
- `POST /hello` принимает форму и строит персональное приветствие;
- на странице видно метод и путь текущего запроса.

## Разница между GET и POST

`GET` обычно используют для чтения данных.
Параметры часто передаются в URL, например:

```text
/?name=Алиса
```

`POST` обычно используют, когда клиент отправляет данные в теле запроса.
В этом примере форма отправляет имя через `POST /hello`.

## Разница между строкой запроса и данными формы

- строка запроса находится в URL и читается через `r.URL.Query()`;
- данные формы приходят в теле запроса и читаются после `r.ParseForm()`.

## Как проходит запрос

### Сценарий со строкой запроса

1. Браузер делает `GET /?name=Алиса`.
2. Handler читает `r.URL.Query().Get("name")`.
3. Сервер вставляет имя в HTML-шаблон.
4. Клиент получает персонализированную страницу.

### Сценарий с POST-формой

1. Браузер отправляет форму на `POST /hello`.
2. Handler вызывает `r.ParseForm()`.
3. Имя читается через `r.FormValue("name")`.
4. Сервер возвращает HTML с персональным приветствием.

## Примеры curl

Query-параметр:

```bash
curl -i "http://localhost:18083/?name=Алиса"
```

POST-форма:

```bash
curl -i -X POST http://localhost:18083/hello \
  -d "name=Алиса"
```

## Где ставить breakpoint

- В handler `GET /` на чтении `r.URL.Query()`.
- В handler `POST /hello` на `r.ParseForm()`.
- На вызове `renderPage`.
