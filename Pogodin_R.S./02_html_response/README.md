# Этап 02. HTML-ответ от сервера

## Учебная цель

Понять, чем HTML-ответ отличается от ответа в виде обычного текста, и увидеть роль шаблонов.

Поток данных:

`браузер -> HTTP-запрос -> handler -> template -> HTML-ответ`

## Как запустить

Из каталога этапа:

```bash
go run .
```

Из корня репозитория:

```bash
make run-02
```

## Какие URL открыть

- `http://localhost:18082/`

## Что демонстрирует приложение

- сервер возвращает не простой текст, а полноценный HTML;
- HTML собирается через `html/template`;
- в шаблон передаются данные из Go-кода;
- страница показывает:
  - заголовок;
  - текущее время сервера;
  - путь запроса.

## Чем HTML отличается от обычного текста

На предыдущем этапе клиент получал обычную строку.
Здесь клиент получает HTML-документ, который браузер умеет интерпретировать
как структуру страницы: заголовки, абзацы, стили и так далее.

## Зачем нужен html/template

Шаблон позволяет:

- держать HTML отдельно от Go-кода;
- подставлять динамические данные;
- не собирать страницу вручную конкатенацией строк.

Для учебного примера это наглядный переход от простого текста к серверному
рендерингу HTML.

## Как проходит запрос

1. Браузер делает `GET /`.
2. Handler формирует структуру данных с текущим временем и путем.
3. `html/template` подставляет данные в HTML-шаблон.
4. Сервер отправляет готовую HTML-страницу.

## Пример curl

```bash
curl -i http://localhost:18082/
```

В ответе будет HTML-код страницы.

## Где ставить breakpoint

- Внутри handler перед созданием `pageData`.
- На вызове `tmpl.Execute`.
