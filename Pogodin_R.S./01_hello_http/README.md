# Этап 01. Hello World по HTTP

## Учебная цель

Понять самый базовый сценарий:

`браузер/клиент -> HTTP-запрос -> handler -> ответ`

Здесь еще нет HTML, форм, JSON и базы данных. Есть только HTTP-сервер, один
маршрут и строка в ответе.

## Как запустить

Из каталога этапа:

```bash
go run .
```

Из корня репозитория:

```bash
make run-01
```

## Какие URL открыть

- `http://localhost:18081/`

## Что такое HTTP-сервер

HTTP-сервер слушает порт, принимает входящие HTTP-запросы и вызывает Go-код,
который умеет сформировать HTTP-ответ.

В этом примере сервер запускается на порту `18081`.

## Что такое handler

Handler — это функция, которая получает:

- `http.ResponseWriter` — через него отправляется ответ клиенту;
- `*http.Request` — в нем лежит входящий запрос.

Именно handler решает:

- какой статус вернуть;
- какой `Content-Type` выставить;
- какой текст или данные отправить обратно.

## Как запрос приходит в код

1. Браузер или `curl` делает `GET /`.
2. `http.ServeMux` находит обработчик для маршрута `GET /`.
3. Вызывается `helloHandler`.
4. Handler пишет строку `"Hello, HTTP!"` в `ResponseWriter`.
5. Клиент получает ответ в виде обычного текста.

## Как отправляется ответ

В коде это происходит через:

- установку заголовка `Content-Type`;
- запись текста в `ResponseWriter`.

## Пример curl

```bash
curl -i http://localhost:18081/
```

Ожидаемый ответ:

```text
HTTP/1.1 200 OK
Content-Type: text/plain; charset=utf-8

Hello, HTTP!
```

## Где ставить breakpoint

- В начале `helloHandler` — видно вход в обработчик.
- На строке записи в `ResponseWriter` — видно момент формирования ответа.

## Что логировать при отладке

- `r.Method`
- `r.URL.Path`
- `r.RemoteAddr`

В примере это уже добавлено в лог сервера.
